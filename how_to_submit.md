## Instructions to Submit Your Test

### 1. Fork the Repository
To begin, you need to **fork this repository** to your GitHub account:

- In the upper right-hand corner of this page, click the "Fork" button to create a copy of this repository under your own GitHub account.

### 2. Clone Your Forked Repository
Next, clone your fork to your local machine:

```bash
git clone https://github.com/your-username/adk-url-shortner-challenge.git
```

### 3. Create a New Branch
Once you've cloned the repo, create a new branch to work on:

```bash
git checkout -b test-submission-{yourname}
```

### 4. Implement the Solution
Start building the URL shortener using the tech stack you're most comfortable with. Be sure to:

Include both frontend and backend components.
Implement URL shortening and retrieval.
Set up a database for storing URL mappings.
Ensure that your application works as expected and is easy to run locally.
### 5. Test Your Solution
Make sure your solution is functional and ready for submission:

Make sure the application can be set up and run locally using the instructions provided in the documentation.
### 6. Push Your Changes
Once you are done, push your changes to your GitHub fork:

```bash
git add .
git commit -m ":rocket: Update - cmplete the adkaora url shortener test"
git push origin test-submission-{yourname}
```

### 7. Submit a Pull Request (PR)
To submit your test, open a pull request (PR) from your fork back to the original repository:

Go to your forked repository on GitHub.
Click the "Compare & Pull Request" button.
In the PR description, please explain:
The key decisions you made.
Any assumptions or simplifications.
Any features you would improve if you had more time.
We will review your PR and get back to you within a few days.

***Documentation Requirements***
In your PR, please ensure that you include:

- Code Documentation: The code should be clean, well-documented, and follow best practices.
- A brief explanation of how to run the application locally.
- An overview of the architecture and any important decisions made during development.
- Suggestions for future improvements or scaling considerations.
